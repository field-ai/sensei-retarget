# Phase 3 — OpEn (Optimization Engine) PANOC Solver

## Goal

Replace IPOPT with a pre-compiled Rust solver generated by [OpEn](https://alphaville.github.io/optimization-engine/).
The cost and constraint expressions are reused from Phase 2 (CasADi symbolic, Pinocchio FK).
The key difference: instead of calling IPOPT at runtime, `opengen` compiles the NLP to Rust once, and the resulting binary is called per-frame via a local TCP socket — expected to cut solve latency from ~18 ms to sub-ms.

---

## Why OpEn over IPOPT / alpaqa

| Property | IPOPT (Phase 2) | OpEn (Phase 3) |
|----------|----------------|----------------|
| Algorithm | Interior-point (2nd order) | PANOC + ALM (1st order) |
| Hessian | Dense, factorized | None — matrix-free |
| Warm start | Dual variables | Primal only |
| Code gen | No — interpreted | Yes — compiled Rust |
| Runtime interface | Python C extension | TCP socket (JSON) |
| Expected latency | ~18 ms | sub-ms (estimate) |
| Embedded target | No | Yes (LLVM → bare metal) |

PANOC avoids matrix factorizations, making it well-suited for moderate-dimensional problems (~30 DoF) where the cost of factorizing the Hessian dominates. The compiled Rust binary eliminates Python interpreter overhead entirely.

## Architecture

```
setup(robot)
  ├── Build Pinocchio model (same as Phase 2)
  ├── Build CasADi symbolic cost + constraint expressions
  ├── Call opengen to generate Rust solver → outputs/open_solver/
  ├── Compile Rust solver (cargo build --release, ~10–30 s, once)
  └── Start TCP server process (OptimizerTcpManager)

solve_frame(targets, q_prev)
  ├── Pack parameters: [root_pos(3), root_rot(4), q_prev(29), ee_targets(...)]
  ├── Send to TCP server → JSON response
  └── Unpack: q(29), converged, solve_time_ms

teardown()
  └── Stop TCP server
```

The Rust solver is compiled once per `setup()` call (or cached if already built). This means the first call to `setup()` is slow (~30 s); subsequent calls in the same session reuse the binary.

## Problem formulation (same as Phase 2)

Decision variables: `q ∈ ℝ²⁹` (joint angles)

Parameters per frame: `p = [root_pos(3), root_rot_wxyz(4), q_prev(29), ee_pos_targets(3×13), ee_rot_targets(9×13)]`

Cost:
```
J(q; p) = Σ_ee [ w_pos · ‖p_fk(q) - p_tgt‖² + w_rot · ‖log(R_fk(q)ᵀ R_tgt)‖² ]
         + w_smooth · ‖q - q_prev‖²
         + w_reg   · ‖q‖²
```

Box constraints (joint limits — handled natively by OpEn as the `U` set):
```
q_lo ≤ q ≤ q_hi
```

ALM constraints (Phase 3b — collision):
```
ground:  foot_z(q) ≥ 0
self:    ‖p_link_i(q) - p_link_j(q)‖² ≥ r²  for key body pairs
```

## Implementation notes

### opengen problem definition

```python
import opengen as og
import casadi as ca

# Decision variable
q = ca.SX.sym("q", 29)

# Parameter vector
p = ca.SX.sym("p", N_PARAMS)

# Cost (reuse Phase 2 symbolic expression)
cost = build_cost(q, p)   # CasADi SX scalar

# Constraints
bounds = og.Rectangle(q_lo.tolist(), q_hi.tolist())

problem = (
    og.Problem(q, p, cost)
    .with_constraints(bounds)
)

meta = og.OptimizerMeta().with_optimizer_name("g1_ik")
build_config = og.BuildConfiguration().with_release_mode(True)
solver_config = og.SolverConfiguration().with_max_outer_iterations(10)

builder = og.OpEnOptimizerBuilder(problem, meta, build_config, solver_config)
builder.build()   # generates + compiles Rust, ~30 s first time
```

### Runtime call

```python
manager = og.tcp.OptimizerTcpManager("outputs/open_solver/g1_ik")
manager.start()

response = manager.call(p_vec.tolist())
if response.is_ok():
    q_sol = np.array(response.get().solution)
```

### Caching the compiled binary

After the first `setup()`, cache the binary path and skip re-compilation if the problem definition hasn't changed (hash the weight vector + joint limits):

```python
cache_key = hashlib.md5(json.dumps({"weights": ..., "limits": ...}).encode()).hexdigest()
if Path(f"outputs/open_solver_{cache_key}").exists():
    # skip build, just start manager
```

## Common failure modes

| Error | Cause | Fix |
|-------|-------|-----|
| `cargo not found` | Rust not installed | `curl https://sh.rustup.rs -sSf \| sh` |
| `opengen.exceptions.BuildError` | Rust compile error in generated code | Check CasADi expression is valid SX (not MX) |
| TCP connection refused | Manager not started | Call `manager.start()` before `manager.call()` |
| `response.is_ok() == False` | PANOC did not converge | Increase `max_outer_iterations`; check parameter range |
| Slow first `setup()` | Rust compile (~30 s) | Expected — binary is cached on subsequent calls |
